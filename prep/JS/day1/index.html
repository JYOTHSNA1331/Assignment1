Execution Context

        Execution context is used to describe how a code works inside.
        It parses the code line by line and stores the variables and functions into the memory.
        Global Execution Context is the first thing that is created when you write JavaScript code.
        Whenever a function gets invoked, called, or executed, a new local execution context gets created.

call stack

        call stack is a data structure that uses the Last In, First Out (LIFO) principle
        The call stack is synchronous.

Declaration of variables in JS

        In JavaScript, we can declare a variable in three different ways by using var, let and const keyword.
        Var
              This keyword is used to declare variable globally. 
              If you used this keyword to declare variable then the variable can accessible globally and changeable also.
              It is good for a short length of codes, if the codes get huge then you will get confused.
              syntax: var a = 5
        let
              This keyword is used to declare variable locally. 
              If you used this keyword to declare variable then the variable can accessible locally and it is changeable as well.
              syntax: let b = 10 
        const
              This keyword is used to declare variable locally.
              const values can't be reassigned. So we should assign the value while declaring the variable.
              syntax: const c = 30

Difference between undefined and not defined in Javascript

        Undefined
             
                Whenever we declare a variable without assigning any value to it, javascript implicitly assigns its value as undefined.
                Ex:
                    let name;
                    console.log(name); //undefined
             

    Not define
    
            A not defined is a variable which is not declared at a given point of time with declaration keyword like var, let or const.
             Ex:
                  console.log(b);
                  b = 5;
            While if we don't use var keyword the output will be:-
                   //Output:- ReferenceError: b is not defined

Hoisting in JavaScript

        The JavaScript engine moves the variable and function declarations to the top of your code. This feature is known as hoisting in JavaScript.
        In variable hoisting keyword var is hoisted and let and const does not allow hoisting.
        Like variables, the JavaScript engine also hoists the function declarations. 
        The JavaScript engine doesn't hoist the function expressions and arrow functions.

Functions in JavaScript

        A Function is a block of code that is designed to perform a task and executed when it is been called or invoked.
       There are 3 ways of writing a function in JavaScript:
       Function Declaration
        function add(a, b) {        
        console.log(a + b);
         }  
         add(2, 3);
        Function Expression
            const add = function(a, b) {
            console.log(a+b);
            }
             add(2, 3);
        Arrow Function
              It is used to shorten the code. 
              Here we do not use the “function” keyword and use the arrow symbol.
             
              let add = (a, b) => a + b;
              console.log(add(3, 2));
